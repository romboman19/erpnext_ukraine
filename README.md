# ERPNext Ukraine

Український бізнес-шар для ERPNext: облік ФОП, управлінська каса/POS, ПРРО,
податковий контроль, українські документи та переклади. Це не форк ERPNext:
застосунок розширює стандартні `Sales Invoice`, склад, платежі й довідники через
DocType, hooks та API.

Репозиторій: `erpnext_ukraine`. Технічне ім'я Frappe app навмисно лишається
`erpnext_ua`, щоб не ламати наявні сайти та міграції.

## Модулі

### UA FOP
- **FOP Profile** — профіль ФОП, привʼязаний до Company: ПІБ, РНОКПП (з перевіркою
  контрольної суми), група ЄП (1/2/3), режим ставки (фіксована / 5% / 3% з ПДВ),
  IBAN, КВЕДи, адреса. Дашборд форми показує актуальні податкові параметри року.
- **UA Tax Parameters** — довідник лімітів доходу і ставок ЄП/ВЗ/ЄСВ по роках і групах.
  Записи на 2026 рік створюються автоматично при міграції (існуючі не перезаписуються).
- **UA KVED** — довідник видів діяльності.

### Податковий календар і контроль лімітів
- **UA Tax Deadline** — дедлайни ЄП/ВЗ/ЄСВ/декларацій по кожному ФОП з урахуванням
  групи; генерація з форми FOP Profile або автоматично (щомісячний scheduler).
- Щоденний scheduler: статуси дедлайнів (Скоро термін / Прострочено) і нагадування
  в Notification Log за 3 дні та при простроченні.
- Монітор доходу (касовий метод: Payment Entry + POS-оплати − повернення) проти
  ліміту групи; алерти при 80/95/100% ліміту; показник на дашборді FOP Profile.

### Друковані форми (Sales Invoice)
- Видаткова накладна (UA)
- Рахунок-фактура (UA)
- Акт виконаних робіт (UA)

Форми підтягують реквізити постачальника з FOP Profile компанії документа.

### UA Fiscal (ПРРО)

- API ДПС станом на 08.08.2025: реєстрація пристрою, каси/зміни, продаж,
  повернення, службові операції, Z-звіт і зв'язок зі стандартним Sales Invoice;
- офіційні XSD, attached ДСТУ-4145/CAdES-E-T через окремий
  `erpnext_ukraine_prro_signer`, криптографічна перевірка квитанцій;
- повний offline-контур: резерв ДПС, CRC32-номери, SHA-256 chain, ліміти
  36/168 годин, пакети до 100 документів/200 KiB і автоматичне відновлення;
- незмінний журнал, ідемпотентність і reconciliation після невизначеної доставки.

### UA POS (каса)

- фізичні каси, допуски працівників і операційні зміни;
- покупюрний перерахунок і незмінний журнал рухів готівки;
- barcode-first кошик `POS Order`, ідемпотентний checkout і Sales Invoice;
- готівкова, карткова та змішана оплата;
- вбудований адаптер банківських POS-терміналів через `pb-pos-gateway`;
- відновлення невизначених карткових операцій через `operation_id`/status;
- фіскалізація через UA Fiscal;
- server-side ESC/POS черга для мережевих принтерів: QR, копії, ретраї та
  `Completed Print Error` без відкочування проведеного продажу.

Архітектура, стани та acceptance-сценарії: [docs/pos/README.md](docs/pos/README.md).

### UA Accounting

- повний український План рахунків за наказом Мінфіну №291 та чинні субрахунки
  Інструкції;
- спрощений План рахунків за наказом №186 із дозволеними операційними
  субрахунками для ERPNext;
- майстер перевірки й застосування лише до компанії без бухгалтерської або
  складської історії; діюча компанія потребує окремої міграції залишків.

Нормативні джерела, обмеження класу 0 і порядок застосування:
[docs/ua-accounting.md](docs/ua-accounting.md).

## Сумісність
- Frappe / ERPNext v16
- Python 3.11+

## Встановлення
```bash
bench get-app https://github.com/romboman19/erpnext_ukraine
bench --site <site> install-app erpnext_ua
bench --site <site> migrate
```

Застосунок постачає централізований український каталог Frappe/ERPNext. Після чистого
встановлення достатньо вибрати `Українська` в користувача або System Settings;
записи `Translation`, fixtures чи окремий скрипт імпорту не потрібні. Для вже
запущеного site після оновлення виконайте `bench --site <site> clear-cache`.

Супровід і перевірка каталогу описані в
[`docs/translations.md`](docs/translations.md).

Production-налаштування, backup/rollback і go-live checklist:
[`docs/production-runbook.md`](docs/production-runbook.md). Відповідність чинному
протоколу ДПС: [`docs/prro-conformance.md`](docs/prro-conformance.md). Вимоги до
фіскальних друкованих форм і дозволеного додаткового footer:
[`docs/receipt-printing-requirements.md`](docs/receipt-printing-requirements.md).

## Межі відповідальності

- Тут: ФОП, каса/POS, банківські термінали в касовому checkout, ПРРО та український compliance.
- У [`erpnext_ukraine_integrations`](https://github.com/romboman19/erpnext_ukraine_integrations):
  зовнішні бізнес-конектори — перевізники, банківські виписки, онлайн-еквайринг,
  маркетплейси, PBX і SMS.
- В [`erpnext_ukraine_prro_signer`](https://github.com/romboman19/erpnext_ukraine_prro_signer):
  ізольований headless-сервіс криптографічного підпису.

## Межі релізу 0.4

Реліз готовий для однієї юридичної особи/ФОП на фізичну касу. Автоматичний
розподіл одного кошика між кількома ФОП, подарункові сертифікати та ЄРПН не
вмикаються приховано: це окремі бізнес-процеси, які потребують затверджених
правил маршрутизації й бухгалтерської моделі. ПДВ у фіскальному чеку
підтримується через явні поля `PRRO Tax Letter/Type`; правильність облікової
схеми ПДВ лишається відповідальністю налаштування ERPNext.
