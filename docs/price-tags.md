# Цінники ERPNext

Модуль `UA Price Tags` централізує формування, друк і аудит роздрібних цінників.
Він не змінює ERPNext core і використовує стандартні `Item`, `Item Price`,
`Price List`, `Warehouse` та складські документи.

## Налаштування

1. Відкрийте **Price Tag Settings**.
2. Вкажіть основний роздрібний прайс-лист.
3. За потреби вкажіть окремий акційний прайс-лист. Його чинна ціна вважається
   простою акцією лише коли вона додатна та нижча за основну.
4. Перевірте звичайний, акційний і пакувальний Print Format.
5. Додайте користувачам роль `Price Tag User`, а відповідальним за налаштування —
   `Price Tag Manager`.

Модуль постачає три макети `40×25 mm`, відтворені за робочими зразками:

- `Цінник звичайний 40x25` — назва, код, велика ціна і штрихкод;
- `Цінник акційний 40x25` — стара та нова ціни, код і штрихкод;
- `Етикетка на упаковку 40x25` — назва, код і збільшений штрихкод без ціни.

Усі вони є звичайними Jinja Print Format для `Price Tag Print Job`. Їх можна
клонувати або замінити макетами Print Designer, не змінюючи логіку пакетів.

## Робочий процес

Кнопка **Друк → Цінники** доступна у:

- Purchase Receipt;
- Stock Entry типу Material Transfer;
- Delivery Note, включно з поверненням;
- Item.

Користувач вибирає рядки, шаблон і режим копій: одна копія, за кількістю
документа або ручна кількість. У режимі `Автоматично` система сама застосовує
звичайний чи акційний макет за чинною ціною. Режим `Етикетка на упаковку`
вибирається окремо й доступний навіть для товару без роздрібної ціни. Система
повторно читає дані на сервері, не довіряє показаній у браузері ціні, та
автоматично розділяє результат за складом і типом шаблону. Захисний ліміт —
1000 копій одного рядка і 5000 цінників в одному пакеті.

Сторінка **Залишки та друк цінників** показує позитивний залишок, чинну ціну,
акційну ціну та дату останнього підтвердженого друку. З неї можна масово створити
пакети для вибраного складу.

## Аудит і повторний друк

`Price Tag Print Job` зберігає назву, штрихкод, одиницю, кількість копій,
звичайну та акційну ціну, термін акції, конкретні `Item Price`, джерело і склад.
Після статусу `Ready` цей знімок незмінний і контролюється SHA-256.

- **Повторити цей знімок** — створює новий пакет із точно тими самими даними.
- Кнопка **Цінники** у вихідному документі — створює новий пакет із актуальними
  цінами на момент натискання.
- **Позначити надрукованим** фіксує користувача й час фізичного друку. Перегляд
  PDF сам по собі цього статусу не змінює.

## Межі першого релізу

- Друк: PDF/браузер, одна етикетка `40×25 mm` на сторінку.
- Проста акція: окремий акційний прайс-лист. Складні `Pricing Rule` (комплекти,
  кількість, сегмент клієнта) навмисно не перетворюються на одну ціну.
- Прямий ZPL/TSPL/QZ Tray друк та автоматичне створення пакетів після погодження
  зміни ціни залишені наступними незалежними етапами.

## Оновлення

Після встановлення нової версії застосунку:

```bash
bench build --app erpnext_ua
bench --site <site> migrate
bench --site <site> clear-cache
```

Перед production-оновленням зробіть backup site і перевірте PDF на фактичному
принтері з масштабом 100% без browser margins.
